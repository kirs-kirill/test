Требования к оформлению документации#

О разметке#

Подробно о применении разметки ReStructuredText можно прочитать здесь или тут.

Структура#

source
├─ conf.py ─ файл настроек документации (авторство, лицензии, расширения, темы)
├─ index.rst ─ основной файл, к нему через toctree присоединяются остальные файлы документации
├─ credits.rst ─ этот файл со схемой и пояснениями
├─ book ─ каталог с разделами
│   ├─ reglament ─ каталог с регламентными документами
│   │   ├─ index.rst ─ основной файл раздела, к нему через toctree присоединяются остальные файлы в этом каталоге
│   │   ├─ 1_inside_regl.rst
│   │   ├─ 2_money.rst
│   │   └─ ...
│   └─ tech
│       ├─ index.rst ─ основной файл раздела, к нему через toctree присоединяются остальные файлы в этом каталоге
│       ├─ 1_uslugi.rst
│       ├─ 2_signal.rst
│       └─ ...
└─ _static ─ каталог со статическими файлами
    └─ images ─ каталог с изображениями
        ├─ reglament
        │   ├─ 4_mass
        │   └─ ...
        └─ tech
            ├─ 1_uslugi
            ├─ 3_test_network
            └─ ...

Теперь рассмотрим структуру повнимательней.

Общее#

Все файлы в каталогах reglament и tech начинаются с цифры, обозначающие место статьи в содержании, т.к. содержание автоматически собирается в алфавитном порядке.

Если необходимо добавить изображение с статью, нужно создать папку с названием статьи в соответсвующем каталоге в _static. Это нужно для упорядочивания хранения статических файлов. Структурно _static повторяет структуру book.

toctree#

В каждом каталоге (кроме _static) есть файл index.rst. Он содержит в себе раздел .. toctree::, эта функция сканирует каталог и, согласно указанным параметрам, собирает книгу из файлов.

Например:

.. toctree::
:maxdepth: 1
:glob:

*

Или:

 .. toctree::
:maxdepth: 1
:numbered:
:caption: Содержание:

book/tech/index
book/reglament/index
credits
Немного о параметрах

  • :maxdepth: - указывает глубину чтения, если указано больше 1, то в содержание будут включены подразделы с основной страницы.

  • :numbered: - нумерует содержание.

  • :glob: - если указан, то не обязательно перечислять нужные файлы для включения в содержание, достаточно указать * и все файлы в каталоге будут автоматически собраны в алфавитном порядке.

Примечание

Комментарии в документе указываются с помощью двух точек ..
